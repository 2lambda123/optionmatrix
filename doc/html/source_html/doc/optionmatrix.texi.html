@node Overview
@chapter Overview
@cindex overview
The OptionMatrix programs are financial derivative calculators
distributed by @url{http://@value{WEBSITE}}.
These calculators are real-time multi-model option chain pricers with
analytics and interactive controls. @command{optionmatrix} is the GTK+ graphical user interface version and @command{optionmatrix_console} is the Curses version (Unix/Linux console, DOS).

Both programs feature: greeks, decimal date to real-date translations, real-date to decimal date translations, real-time time bleeding, configurable option expiration date engines, calendars, strike control systems, tickers and over 135 option models. @command{optionmatrix} also supports: spreads, bonds, term structures, cash flow editing, source code viewing and text exporting.

@cindex authors
@cindex Anthony Bradford
@cindex Bj@o{}rn Augestad
@cindex Metaoptions
@cindex Bernt Arne @O{}edegaard
@cindex Financial Numerical Recipes in C++

Model authors include:  Bj@o{}rn Augestad (Metaoptions), @url{http://@value{WEBSITE}, Anthony Bradford}, @url{http://www.google.com/search?q=Bernt+Arne+Oedegaard,Bernt Arne @O{}edegaard} (Financial Numerical Recipes in C++) and Seth Pinsky. The code base can be easily extended to include new models.

@section Usage Options
@cindex usage

The options for running the OptionMatrix programs are:

@example
optionmatrix @var{option} @dots{}
@end example

@example
optionmatrix_console @var{option} @dots{}
@end example

@cindex curses
With no options, @command{optionmatrix} launches the GTK+ (GUI) version of OptionMatrix and @command{optionmatrix_console} launches the Curses (console, DOS) version.

@command{optionmatrix} and @command{optionmatrix_console} support the following options:

@table @option

@cindex help
@item --help
@itemx -h
@opindex --help
@opindex -h
Print an informative help message on standard output and exit.

@item --version
@itemx -v
@opindex --version
@opindex -v
Print the version number and licensing information of OptionMatrix on
standard output and exit.

@item --list
@itemx -l
@opindex --list
@opindex -l
@cindex Listing models
@cindex Models listing
List the names of all financial models to standard output and exit.

@item --source
@itemx -s
@opindex --source
@opindex -s
@cindex Source code
List source code file names of models not accessible by the application to standard output and exit. If a model's source code is accessible the @command{optionmatrix} program can display source code from the running application.

@item --debug
@itemx -d
@opindex --debug
@opindex -d
Force creation of debug log file @file{optionmatrix.log}

@item --price
@itemx -p
@opindex --price
@opindex -p
Iterate through and test all models to standard output and exit.

@item --quiet
@itemx -q
@opindex --quiet
@opindex -q
Iterate through and test all models in quiet mode and exit.

@item --Directory
@itemx -D
@opindex --Directory
@opindex -D
Set model source code directory so GTK+ version of the program can display.

@item --model
@itemx -m
@opindex --model number
@opindex -m number
Time test a specific model number to standard output and exit. 200000 executions of model are made and timed by default.

@item --iterate number
@itemx -i number
@opindex --iterate number
@opindex -i number
Set number of executions a model will be time tested for (default 200000)

@end table
The options @option{--debug} and @option{-d} are the only options used by both @command{optionmatrix} and @command{optionmatrix_console} applications. The option @option{--Directory} is exclusive to @command{optionmatrix}. The remaining command line options do not launch an interactive application and only create standard output and exit.

@section Main Screen

@cindex GTK+
@cindex optionmatrix.log
@cindex Option chain
@cindex Black-Scholes
@cindex Time bleeding
Running @command{optionmatrix} with no arguments or just the @option{--debug}, @option{-d} options will invoke the OptionMatrix GTK+ Graphical Application. The @option{--debug} or @option{-d} options will cause @command{optionmatrix} to create the debug log, @file{optionmatrix.log}, in the users home directory or the current working directory of execution. This log file will be populated with diagnostic information while the program is executing.

The initial screen will appear. The default model is Black-Scholes. The option chain on display is being re-calculated every second. Price changes are the result of time bleeding between the current time and the expiration time.

@section Date Engine
@cindex Date engine

@*
@ifdocbook

@end ifdocbook

@cindex expiration
@cindex Exchange traded options
The Date Engine will map option expirations to a default of 3rd Friday+1 @@11:59AM going into the future. This is the standard expiration for U.S. Exchange Traded Options. The Date Engine is configurable and can make any re-occuring date used by the options industry.

The option chain's pricing will be updated as time decreases between the current time and the expirations.

@section Running
@cindex curses
@cindex optionmatrix.log
Running @command{optionmatrix_console} with no arguments or just the @option{--debug}, @option{-d} options will invoke the OptionMatrix Curses Application.  The Curses Application is a text application with no graphical user interface. The @option{--debug} or @option{-d} options will cause @command{optionmatrix_console} to create the debug log, @file{optionmatrix.log}, in the users home directory or the current working directory of execution. This log file will be populated with diagnostic information while the program is executing.

@cindex UNIX
The Curses Application typically runs from a UNIX like console or DOS prompt, it can be associated with a start menu or desktop icon. The Curses Application responds to keyboard events. @xref{Graphical Application optionmatrix}, for optionmatrix with a graphical user interface.

@cindex Bug reporting
@cindex problems
@cindex Reporting bugs

To report bugs, suggest enhancements or otherwise discuss optionmatrix,
please send electronic mail to @email{info@@@value{WEBSITE}}.

@cindex Checklist for bug reports
For bug reports, please include enough information for the maintainers
to reproduce the problem.  Generally speaking, that means:

@itemize @bullet
@item The version numbers of optionmatrix (which you can find by running
      @w{@samp{optionmatrix --version}}) and any other program(s) or
      manual(s) involved.
@item Hardware and operating system names and versions.
@item The contents of any input files necessary to reproduce the bug.
@item The expected behavior and/or output.
@item A description of the problem and samples of any erroneous output.
@item Options you gave to @command{configure} other than specifying
      installation directories.
@item Anything else that you think would be helpful.
@end itemize

When in doubt whether something is needed or not, include it.  It's
better to include too much than to leave out something important.

@cindex Patches, contributing
Patches are welcome; if possible, please make them with @samp{@w{diff
-c}}. Please follow the existing coding style.
